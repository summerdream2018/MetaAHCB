# MetaAHCB User Manual

## Usage

## 1. Sample Grouping

Before running MetaAHCB, samples should be grouped according to their expected biological similarity. Samples within the same group will be processed together, including cross-sample read mapping and coverage-based binning.

For example, six samples are available:  

- `sample01`: sediment sample
- `sample02` and `sample03`: coastal seawater samples
- `sample04`, `sample05`, and `sample06`: offshore seawater samples

The third-generation sequencing (TGS) reads generated by Cyclone are stored as follows. Multiple sequencing files may exist for the same sample.  

`./cyclone/pathway01/sample01.fq.gz`
`./cyclone/pathway02/sample01.fq.gz`
`./cyclone/pathway03/sample01.fq.gz`
`./cyclone/pathway04/sample02.fq.gz`
`./cyclone/pathway05/sample03.fq.gz`
`./cyclone/pathway06/sample04.fq.gz`
`./cyclone/pathway07/sample04.fq.gz`
`./cyclone/pathway08/sample04.fq.gz`
`./cyclone/pathway09/sample05.fq.gz`
`./cyclone/pathway10/sample05.fq.gz`
`./cyclone/pathway11/sample06.fq.gz`

The corresponding second-generation sequencing (NGS) paired-end reads are organized as:  

`./NGS_PE/pathway01/sample01_1.fq.gz`
`./NGS_PE/pathway01/sample01_2.fq.gz`
`./NGS_PE/pathway02/sample02_1.fq.gz`
`./NGS_PE/pathway02/sample02_2.fq.gz`
`./NGS_PE/pathway03/sample03_1.fq.gz`
`./NGS_PE/pathway03/sample03_2.fq.gz`
`./NGS_PE/pathway04/sample04_1.fq.gz`
`./NGS_PE/pathway04/sample04_2.fq.gz`
`./NGS_PE/pathway05/sample05_1.fq.gz`
`./NGS_PE/pathway05/sample05_2.fq.gz`
`./NGS_PE/pathway06/sample06_1.fq.gz`
`./NGS_PE/pathway06/sample06_2.fq.gz`

Based on ecological characteristics, these six samples can be divided into three groups:

| Group     | Description             | Samples                            |
| --------- | ----------------------- | ---------------------------------- |
| `group01` | Sediment group          | `sample01`                         |
| `group02` | Coastal seawater group  | `sample02`, `sample03`             |
| `group03` | Offshore seawater group | `sample04`, `sample05`, `sample06` |

Each group will be processed independently and will generate a separate set of MAGs.

## 2. Installation and Initialization

Assume the MetaAHCB installation directory is:

`./user1/software/MetaAHCB/`

and the project directory is:

`./user1/metaG_project_01/`

Copy the MetaAHCB program directory to the user-defined software directory:

`cp -rf /path/to/MetaAHCB/ ./user1/software/`

After installation, the main program and scripts should be located at:

`./user1/software/MetaAHCB/pipeline.sh`

`./user1/software/MetaAHCB/scripts/`

Before running the pipeline, modify the following settings in `pipeline.sh`.

(1) Script directory  

Set:  

`scripts_DIR`

to the absolute path of the scripts/ directory.

Example:

`scripts_DIR=./user1/software/MetaAHCB/scripts/`

(2) Pipeline parameters  

Modify workflow parameters after:  

`### Set parameters`  

The configurable parameters include read processing, clustering, correction, assembly, and binning settings.  

3. Software paths and environments  

Modify software paths and environment activation commands after:  

`### Software directory and activate`  

Ensure that all required software paths and conda environments are correctly specified before execution.
